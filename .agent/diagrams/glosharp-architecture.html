glosharp architecture

twoslash for C# — system design overview

Overview

GloSharp extracts rich symbol metadata from C# code using Roslyn, producing JSON that rendering integrations consume. The system splits into a .NET core that does the heavy lifting, a CLI interface that produces JSON, and thin JS integrations for Shiki, Expressive Code, and standalone HTML.

Core Components
glosharp Core GloSharp.Core

The .NET library that does the heavy lifting. Parses C# source via Roslyn, creates a compilation with appropriate references, walks the syntax tree, and extracts symbol metadata at marker positions.

Key APIs

  • CSharpSyntaxTree.ParseText() parse source
  • CSharpCompilation.Create() compilation context
  • SemanticModel.GetSymbolInfo() resolve symbols
  • ISymbol.ToDisplayString() format type info
  • Compilation.GetDiagnostics() compile errors

Responsibilities

  • Parse and process glosharp marker syntax
  • Walk syntax tree for metadata extraction
  • Produce structured GloSharpResult output
  • Report diagnostics for CI failure
CLI & Bridge
glosharp CLI dotnet tool

The command-line interface. Resolves project context, calls core library, outputs JSON to stdout. Exits non-zero on compile errors for CI.

# Process a file with project context glosharp process src/Example.cs --project src/Example.csproj # Extract a specific region glosharp process src/Example.cs --region getting-started # Verify all snippets compile (CI) glosharp verify samples/
Node.js Bridge npm: glosharp

Spawns the CLI as a child process, parses JSON output, provides a typed TypeScript API. Caches results during builds.

import { createGloSharp } from 'glosharp' const tw = createGloSharp() const result = await tw.process({ file: 'src/Example.cs', project: 'src/Example.csproj' }) // result.hovers, result.errors, result.code
Rendering Integrations
Shiki Transformer

@glosharp/shiki

Follows the @shikijs/twoslash pattern. Runs glosharp in the preprocess hook, injects HAST nodes in the root hook.

  • Covers VitePress, Nuxt, any Shiki user
  • HAST nodes with CSS classes
Expressive Code Plugin

@glosharp/expressive-code

Uses EC's annotateCode hook with custom GloSharpHoverAnnotation. Richest feature set with theme-aware styles and frames.

  • Covers Astro/Starlight (EC built-in)
  • EC annotations + popup rendering
Standalone Renderer

glosharp render

Self-contained HTML with inline CSS. Uses CSS anchor positioning for hover tooltips. No JS dependencies in output.

  • Covers Hugo, Jekyll, anywhere
  • Zero external dependencies

Design Principles
JSON boundary is the contract Everything downstream of the CLI is a thin adapter. The core outputs JSON; integrations consume it. Clean separation of .NET and JS worlds.
Fail loud in CI Compile errors should break the build. If the code in your docs doesn't compile, the doc build fails. No silent rot.
Framework-agnostic core The .NET library knows nothing about Shiki, Expressive Code, or any rendering framework. It produces data, not HTML.
Build-time only No runtime JavaScript required in rendered output. CSS anchor positioning handles hover tooltips. Ship zero JS to readers.
